# Lukas Lechner Brand System

## Positioning

This brand should signal practical AI engineering education for ambitious technical learners.

The identity should feel:

- sharp
- modern
- calm
- technical
- premium
- instructional

It should not feel:

- generic SaaS
- flashy sci-fi
- cartoonish
- overdesigned
- corporate gradient-heavy

## Brand Architecture

- Master brand: `Lukas Lechner`
- Sub-brand: course and product names sit underneath the master brand
- Default course lockup:
  - `Lukas Lechner`
  - `AI Engineering Foundations`

Use the personal brand as the anchor across:

- website
- slide decks
- YouTube thumbnails
- LinkedIn and X banners

## Visual Principles

1. Use structure instead of decoration.
2. Use one vivid accent color only.
3. Keep layouts spacious and readable.
4. Make technical content feel premium, not noisy.
5. Prefer geometric and editorial forms over illustrative clichés.

## Color System

- `Ink 950`: `#0B1020`
- `Ink 800`: `#1A2236`
- `Slate 500`: `#6B7280`
- `Mist 100`: `#E7EDF7`
- `Paper`: `#FAFBFD`
- `Signal Blue`: `#2F6BFF`
- `Signal Blue Light`: `#8FB2FF`

### Usage

- Use `Paper` and `Mist 100` for slide and page backgrounds.
- Use `Ink 950` for covers, banners, and hero modules.
- Use `Signal Blue` sparingly for emphasis, underline accents, labels, and CTA moments.
- Do not add purple, neon green, or extra accent families unless the system is intentionally revised.

## Typography

- Display: `Space Grotesk`
- Body/UI: `IBM Plex Sans`
- Code: `IBM Plex Mono`

### Rules

- Use `Space Grotesk` for titles, key statements, and high-impact labels.
- Use `IBM Plex Sans` for paragraphs, bullets, UI labels, and supporting copy.
- Use `IBM Plex Mono` only for code, schemas, and technical metadata.
- Keep tracking slightly tighter on large display headlines.

## Layout System

### Slides

- Base canvas: `1600x900`
- Keep a generous outer margin.
- Use strong left alignment.
- Avoid crowded two-column layouts unless the content is truly balanced.

### Web

- Base desktop system: `12-column grid`
- Use the same spacing rhythm as slides.
- Let the website feel like the slide system expanded into an interface.

## Logo System

### Primary wordmark

- Text: `Lukas Lechner`
- Use display typography and a restrained signal-blue underline

### Course lockup

- Wordmark on first line
- Course name below in smaller supporting type

### Monogram

- `LL`
- Dark square background
- Blue signal bar near the top
- White initials centered

### Rules

- Default to dark text on light backgrounds
- Use reversed white-on-dark versions for banners and cover slides
- Use the monogram for favicons, placeholders, and secondary avatar-safe surfaces
- Use your photo for primary social avatars

## Icon Style

- Base grid: `24x24`
- Stroke weight: `2`
- Round line caps and joins
- Outline-first forms
- Tiny accent fills are allowed in `Signal Blue`

Do not use:

- 3D lighting
- filled cartoon icons
- mixed stroke weights
- tiny visual details that disappear at small sizes

## Generated Image Style

Generate images that look like:

- abstract geometric systems
- layered panels
- grid fields
- directional lines
- sparse data-like textures
- calm technical surfaces

Do not generate:

- robot heads
- glowing brains
- circuit-board wallpaper
- humanoids
- hologram hands
- generic cyberpunk skylines

Every generated image should be normalized after generation:

1. crop to a consistent composition
2. reduce unnecessary saturation
3. bring colors back to the `Ink + Signal` palette
4. add the same overlay treatment if needed
5. export only the approved version

## Surface Rules

### Slides

- Covers and section dividers can be dark
- Teaching slides should usually be light
- Code slides should use dark code panels on light surfaces
- Diagrams should use white nodes on light backgrounds with blue connectors

### Social

- Keep banners dark for impact
- Maintain large type and safe zones
- Avoid clutter because social crops are aggressive

### Website

- Hero sections can be dark or light
- Keep content modules mostly light for readability
- Maintain the same type hierarchy used in decks
