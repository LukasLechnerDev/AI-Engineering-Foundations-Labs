# Outline

## Learning goals
- Use Langfuse for basic observability while the MVP is still being built.
- Understand how traces help inspect failures, compare runs, and debug changes.
- See how prompt management and versioning fit into the working project.

## Notes
- This module stays in the MVP path because students benefit from basic tracing before deployment.
- Keep the focus on observability: traces, prompts, latency, token usage, and debugging bad runs.
- Langfuse-based evaluation workflows do not belong in this module and should come later.
- Emphasize transferable observability concepts, not just one vendor UI.
