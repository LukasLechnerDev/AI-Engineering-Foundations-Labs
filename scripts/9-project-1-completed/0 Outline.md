# Outline

## Learning goals
- Recognize that the `ai-engineer-job-digest` MVP is complete at this point.
- Reflect on what the raw SDK approach made easier to understand.
- Prepare students for the optional post-MVP modules that come next.

## Notes
- This module marks the end of the core MVP path.
- Everything after this point is an extension: production hardening, Langfuse-based eval workflows, framework comparisons, RAG, and agents.
- Students should leave this module with a working project before the course adds more moving parts.

## Script
- Recap the finished system end to end.
- Highlight what worked well in the raw SDK approach.
- Point out where abstractions might help in a larger codebase.
- Frame the rest of the course as reflective bonus material after the fundamentals are already clear.
- Bridge into the next post-MVP modules:
  - productionizing the system
  - using Langfuse for later evaluation workflows on top of the local eval suite
  - comparing the raw SDK with frameworks such as LangChain or PydanticAI
