# 10 - No-code tools vs custom AI app

## Learning goals
- Understand what people mean by no-code automation tools like `n8n` and `Zapier`.
- See why a simple automation can be enough for some AI workflows.
- Understand why this course builds a custom Python app instead.

## Notes
- Keep this lecture short and practical.
- The goal is not to teach `n8n` or `Zapier`.
- The goal is to give students a simple decision rule before they build the project in Python.

### No Code Tools Benefits and drawbacks
- No code tools obviously don't need any programming experience
- Many tools available, like n8n
- Limited to the features provided by the tool
- These tools are quite capable

### With Code Benefits and drawbacks
- Requires basic programming knowledge
- Can use any programming language
- Flexible
- Full control, No Limits

## Script
- Mention that tools like `n8n` and `Zapier` let you connect services and automate straightforward workflows without writing much code.
- Explain that they are a good fit when the flow is simple, the steps are easy to see, and speed matters more than flexibility.
- Contrast that with our project: we want more control over prompts, data structures, testing, debugging, versioning, and deployment.
- Explain that this is why the course builds a custom Python app instead of a no-code workflow.
- Give the rule of thumb:
  - use no-code tools for simple internal automations
  - use custom code when the product logic starts to matter
- Close by saying that learning Python here is not just about building this one project.
- It is about learning how AI features turn into real software.
