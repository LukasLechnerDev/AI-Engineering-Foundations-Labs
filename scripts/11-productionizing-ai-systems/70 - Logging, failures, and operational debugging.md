# 70 - Logging, failures, and operational debugging

## Learning goals
- Debug AI systems when something goes wrong in production.
- Distinguish between useful logs and noisy logs.
- Prepare students for a later observability module without making external tooling mandatory.

## Notes
- This lesson should stay tool-agnostic.
- Students should first understand what to log in code before using a platform like Langfuse.
- The follow-up observability module can then show the same debugging workflow with traces.

## Script
- useful logs vs noisy logs
- what to log around prompts, tool calls, and outputs
- failure categories for AI systems
- debugging bad runs with logs first, then observability traces later when available
- turning recurring failures into eval cases or product fixes
