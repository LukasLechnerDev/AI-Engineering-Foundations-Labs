# 10 - Deferred topics

## Deferred from the MVP
- parameters and parameter count
  - useful later when discussing model tradeoffs
- top-k and top-p
  - advanced sampling details, not needed for the first project
- open source vs open weight
  - useful ecosystem knowledge, but not required to ship the MVP
- token pricing and cost calculators
  - useful later when discussing production tradeoffs
- context windows and max output tokens
  - useful when model limits become relevant in practice
- knowledge cutoff
  - useful when discussing model limitations
- reasoning vs non-reasoning models
  - useful later when students compare models

## Why this is separate
- The MVP needs a simple mental model, not a full survey of LLM concepts.
- Students can build the first project successfully without this extra theory.
