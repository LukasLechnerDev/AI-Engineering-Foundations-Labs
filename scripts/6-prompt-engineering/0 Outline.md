# Outline

## Teaching goal
- Students should learn how to improve weak but working prompts inside the `ai-engineer-job-digest` project.
- This module should feel like practical iteration on real tasks, not a survey of prompt tricks.

## Where this fits
- Students already know how to make model calls from the previous module.
- Now the question is no longer "How do I call the model?"
- The question is "How do I get better results for my task?"

## What to cover
1. Prompt engineering as iterative improvement
2. Start from a weak baseline prompt in the project
3. Make the task and decision criteria more explicit
4. Add clear constraints and success criteria
5. Use examples for ambiguous cases
6. Break larger tasks into smaller promptable steps
7. Apply this to project tasks like classification, extraction, and summarization
8. Iterate on prompts with a small set of representative examples

## Project fit
- This module starts when the `ai-engineer-job-digest` output quality becomes the bottleneck.
- A good first anchor example is the job-posting classification prompt.
- The same iteration pattern can later be reused for extraction and summarization steps.
- The goal is to make the project output noticeably better with simple, testable prompt changes.
