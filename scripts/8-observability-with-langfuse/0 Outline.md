# Outline

## Teaching goal
- Students should learn how to see what their AI system is doing before they try to optimize or harden it.

## What to cover
- why observability matters for AI systems
- tracing a multi-step pipeline
- prompt and version inspection
- token usage visibility
- latency visibility
- comparing runs across prompt or model changes
- inspecting tool calls and intermediate steps
- using traces to debug poor outputs

## Keep out of this module
- retries and fallbacks
- reliability strategy
- model routing
- deployment concerns
