# Outline

## Teaching goal
- This module is where students learn how to make an AI feature usable in the real world after they already understand prompting, evals, and observability.

## What to cover
1. From prototype to product
2. Model choice and routing
3. Cost and latency tradeoffs
4. Handling rate limits and retries
5. Defensive parsing and output validation
6. Fallbacks and graceful degradation
7. Logging, failures, and operational debugging

## Why this module exists
- These topics were removed from the model API module on purpose.
- They belong together as production engineering concerns.
